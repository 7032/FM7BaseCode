# DEVICE.md — 機種判別と入力デバイス (FM 音源 / ジョイスティック)

実行中の機種を判別し、 入力デバイス (FM 音源・ジョイスティック) を扱う小さな C モジュールです。 実装は [src/c_device.c](../src/c_device.c) / [src/c_device.h](../src/c_device.h)。 起動時に「いま動いているのは FM-7 か FM77AV 以降か」「FM 音源は載っているか」を調べ、 メインループで「ジョイスティックがどう倒れているか」を読む、 という用途を想定しています。

> 仕様の根拠は F-BASIC 文法書 / ユーザーズマニュアル等の公式公開資料に基づきます。 FM 音源 (OPN) のアクセスは PSG ([SOUND.md](SOUND.md)) と同じ BDIR/BC1 プロトコルで、 ポートだけが異なります。

## 1. API 一覧

| 関数 | 役割 |
| --- | --- |
| `unsigned char GetMachineType(void)` | 動作中の機種・機能を判別し `FEAT_*` ビットマスクを返す |
| `const char *GetMachineName(void)` | 機種名の文字列を返す ("FM-7" 等) |
| `unsigned char HasFMSound(void)` | FM 音源 (OPN) 搭載なら 1、 非搭載なら 0 |
| `unsigned char JoyStick(int nID)` | ジョイスティックの状態を読む (nID=0:1P / 1:2P) |

## 2. GetMachineType() — 機種・機能の判別

戻り値は、 検出した**機能ビット (`FEAT_*`) の OR** です。 「機種」を 1 つの番号で返すのではなく、 「どの機能・オプションを備えているか」をビットで表し、 機種はその組み合わせとして読み取ります。

| 定数 | 値 | 意味 | 区分 |
| --- | --- | --- | --- |
| `FEAT_FMSOUND` | `0x01` | FM 音源 (OPN) が使える (FM-7=後付けカード / AV=内蔵) | 周辺 |
| `FEAT_AV` | `0x08` | **FM77AV 系** (アナログ表示 / ALU / ジョイスティック / OPN 内蔵) | 世代境界 |
| `FEAT_2DD` | `0x10` | 2DD 世代 (AV20 / AV40 以降) | AV 世代 (予約) |
| `FEAT_400LINE` | `0x20` | 400 ライン (AV40 系 / 77L4) | AV 世代 (予約) |
| `FEAT_DMAC` | `0x40` | DMAC (AV20EX / 40EX / 40SX) | AV 世代 (予約) |
| `FEAT_26MAN` | `0x80` | 26 万色 (AV40EX / 40SX 級) | AV 世代 (予約) |

`FEAT_FMSOUND` (`0x01`) が周辺オプション、 `FEAT_AV` (`0x08`) が FM77AV 系の世代境界、 上位 4 bit が AV の世代を表します (`0x02` / `0x04` は将来の周辺オプション用に空けてあります)。 「FM77AV 以降か?」は `(GetMachineType() & FEAT_AV)` で判定できます。 値の大きさはおおむね機種の世代順に対応します (例: 素の FM-7 `0x00` < FM77AV `0x09` < AV20 `0x19` < AV40 `0x39` < AV40EX `0xF9`)。

### 本テンプレでの実装状況 (重要)

ビットの**定義**と、 実機で**確実に検出できるか**は別問題です。 本テンプレが実際に検出して立てるのは、 実機で確実かつ非破壊に判別できる次の 2 つだけです。

- **`FEAT_FMSOUND` (`0x01`)**: FM 音源 (OPN) レジスタの読み書きで判定 ([4 章](#4-hasfmsound--fm-音源-opn-搭載判定) と同じ仕組み)。 FM-7 + 後付けカードでも FM77AV でも立ちます。
- **`FEAT_AV` (`0x08`)**: 下記「判別方針」のとおり、 FM77AV 専用の拡張サブシステムへ切り替えられるかで判定。

残りのビットは **常に 0 (予約)** です。 上位 4 bit (`FEAT_2DD` / `FEAT_400LINE` / `FEAT_DMAC` / `FEAT_26MAN`) は将来の AV 世代細分類のための予約で、 `0x02` / `0x04` は将来の周辺オプション用の空きです。 したがって現状の実測値は **FM-7 = `0x00` / FM-7 + FM 音源 = `0x01` / FM77AV = `0x09` (`FEAT_FMSOUND | FEAT_AV`)** になります。

### 判別方針 (FEAT_AV)

FM77AV 系は、 サブ CPU の動作プログラム (サブシステム) を **FM77AV 専用の拡張版へ切り替えられる**のが特徴です。 素の FM-7 にはこの切替の仕組みが無く、 常に FM-7 互換サブシステムのままです (320x200 アナログ表示などの FM77AV 専用機能は、 この拡張サブシステム側にあります)。

そこで「**拡張サブシステムへ切り替えられるか**」を、 切替を試みてサブシステムの応答が変化するかどうかで観測して判別します。 切り替われば FM77AV 以上、 切り替わらなければ素の FM-7 です。 判別のあとはサブシステムを FM-7 互換の状態へ**必ず戻す**ので、 通常の描画 ([SUBPROGRAM.md](SUBPROGRAM.md)) はそのまま使えます。

## 3. GetMachineName() — 機種名文字列

`(GetMachineType() & FEAT_AV)` が立っていれば `"FM77AV"`、 そうでなければ `"FM-7"` を返します (AV のサブ機種は本テンプレでは区別しません)。 画面表示やデバッグ出力にそのまま使えます。

## 4. HasFMSound() — FM 音源 (OPN) 搭載判定

FM 音源 (OPN) が載っていれば 1、 載っていなければ 0 を返します。 **FM77AV 以降は標準搭載**のため必ず 1 になります。 FM-7 では FM 音源は拡張扱いなので、 実際に載っているかどうかを判定する必要があります。

判定は **FM 音源レジスタに値を書いて読み戻し、 保持されているか**で行います。 読み書き可能なレジスタへ既知の値を書き込み、 読み戻して一致すれば搭載、 一致しなければ非搭載とみなします (判定後は元の値に戻します)。

## 5. JoyStick(nID) — ジョイスティックの状態

`nID=0` で 1P、 `nID=1` で 2P のジョイスティックを読みます。 入力は **FM 音源 (OPN) の I/O ポート経由**で取得するため、 **FM 音源を搭載しない素の FM-7 では読み出せず `0xFF` を返します**。

戻り値は次のビットの組み合わせです。 **アクティブ Low** で、 **0 = 押下 / 1 = 非押下**。 未接続・未搭載のときは全ビット 1 の `0xFF` になります。

| 定数 | 値 | 意味 |
| --- | --- | --- |
| `JOY_UP` | `0x01` | 上 |
| `JOY_DOWN` | `0x02` | 下 |
| `JOY_LEFT` | `0x04` | 左 |
| `JOY_RIGHT` | `0x08` | 右 |
| `JOY_TRIG_A` | `0x10` | トリガ 1 (A ボタン) |
| `JOY_TRIG_B` | `0x20` | トリガ 2 (B ボタン) |

押下判定は「該当ビットが 0 か」で行います (アクティブ Low に注意)。

```c
if ((JoyStick(0) & JOY_UP) == 0) {
    /* 1P の上が押されている */
}
```

## 6. 使用例

起動時に機種と FM 音源の有無を見て分岐し、 メインループでジョイスティックを読む例です。

```c
#include "c_device.h"

void main(void)
{
    unsigned char joy;

    /* 起動時: 機種と入力環境を確認 */
    if (GetMachineType() & FEAT_AV) {
        /* FM77AV 以降 (= FM 音源は標準搭載) */
    }
    if (!HasFMSound()) {
        /* FM 音源が無い → ジョイスティックは使えない */
    }

    for (;;) {
        joy = JoyStick(0);          /* 1P を読む */

        if ((joy & JOY_LEFT)  == 0) { /* 左へ */ }
        if ((joy & JOY_RIGHT) == 0) { /* 右へ */ }
        if ((joy & JOY_TRIG_A) == 0) { /* A ボタン */ }

        /* ... 描画・フレームペーシングなど ... */
    }
}
```

## 7. 注意

- **ジョイスティックは FM 音源 (OPN) 前提**です。 FM 音源を搭載しない素の FM-7 では `JoyStick()` は常に `0xFF` を返すため、 ゲーム側ではキーボード入力 ([SUBPROGRAM.md](SUBPROGRAM.md) のキー入力) など別の手段を併用してください。
- AV40 / AV40EX 固有の機能 (DMAC 等) は本テンプレの対象外です。 対応する上位ビット (`FEAT_2DD` / `FEAT_400LINE` / `FEAT_DMAC` / `FEAT_26MAN`) は予約で、 現状は常に 0 です。
- **`GetMachineType()` は起動シーケンスの早い段階で呼ぶこと**。 FM77AV では判別中にサブシステムを一時切替 (= サブ CPU リセット) するため、 サブプログラムや sprite/背景タイルを sub へ転送した**後**に呼ぶと、 転送済みの内容が失われます。 機種判定が必要なら、 `subprog_init()` など sub への転送より**前**に 1 回呼んで結果を保持しておくのが安全です (判定後はサブシステムが FM-7 互換状態に戻るので、 その後の転送・描画は通常どおり)。
