# SOUND.md — PSG (AY-3-8910) サウンド

効果音と BGM を **PSG (AY-3-8910)** で鳴らします。実装は [src/c_sound.c](../src/c_sound.c) / [src/c_sound.h](../src/c_sound.h)、メインループからの呼び出しは [src/c_main.c](../src/c_main.c)。

> ⚠️ 音程の `period` 値や PSG クロックは機種/エミュで差が出ることがあります。音高がずれる場合は §5 の表の値を調整してください（**実機/エミュで要確認**）。

## 1. 概要 — サウンドは「メイン側」で完結する

- FM-7 のサウンドは **AY-3-8910 (PSG)**：**矩形波 3 チャンネル + ノイズ 1 系統**。
- PSG は **メイン CPU 側の I/O** (`$FD0D` / `$FD0E`) で叩きます。グラフィック・キー入力を担うサブシステム（HALT / 共有 RAM / subprog）とは**完全に独立**しているので、サウンドは `c_main` 側だけで完結し、あの繊細なサブ系統に一切干渉しません。
- 発音タイミングは**フレーム同期**：メインループ先頭のフレームペーシング（= メインタイマ IRQ 約2ms 刻みの deadline。[TIMER.md](TIMER.md)）の直後に `sound_tick()` を 1 回呼ぶだけ。BGM の音長カウントも SE の減衰/リリースもこのティックで進みます。

## 2. PSG ハードウェア

| 項目 | 値 |
| --- | --- |
| チップ | AY-3-8910 (PSG) |
| 音源 | 矩形波 3ch (A/B/C) + ノイズ 1 系統 |
| 音量 | 各 ch 16 段階 (0-15) + ハードウェアエンベロープ |
| 周波数 | PSGクロック ÷ (16 × period) |

### アクセス手順（`$FD0D` BDIR/BC1 + `$FD0E` データバス）

PSG レジスタへの書込みは「アドレスをラッチ → データを書く」の 2 段。`$FD0E` にレジスタ番号や値を載せ、`$FD0D` の下位 2 bit で動作を指定します。

```
レジスタ番号 → $FD0E      ; データバスにアドレス
$03          → $FD0D      ; アドレスラッチ (BDIR=1, BC1=1)
$00          → $FD0D      ; インアクティブ (ラッチ確定)
書き込む値   → $FD0E      ; データバスに値
$02          → $FD0D      ; データ書込み (BDIR=1, BC1=0)
$00          → $FD0D      ; インアクティブ (書込み確定)
```

これを `c_sound.c` の `psg_write(reg, val)` が行います（メモリマップ I/O。共有 RAM の `SUB_CMD_REG` と同じく絶対番地ポインタで書く流儀）。

### 主なレジスタ

| reg | 役割 |
| --- | --- |
| R0/R1 | ch A tone period (fine/coarse、 12bit) ※本テンプレ未使用 |
| R2/R3 | ch B tone period — 歩行音 |
| R4/R5 | ch C tone period — BGM |
| R6 | ノイズ周期 (5bit) — ボール発射音 |
| R7 | mixer (0=有効。 bit0-2=tone A/B/C, bit3-5=noise A/B/C) |
| R8/R9/R10 | ch A/B/C 音量 (0-15、 bit4=エンベロープ) |
| R11-R13 | エンベロープ周期/形状 (本テンプレ未使用、 §8 拡張) |

## 3. チャンネル割り当てとフレーム駆動

競合・優先度を持たない**単純構成**：3 チャンネルを用途ごとに固定します。

| ch | 用途 | 種別 |
| --- | --- | --- |
| A | **SE: ボール発射音** | ノイズ・バースト + 減衰 |
| B | **SE: 歩行音** | 短い低音ブリップ (矩形波) |
| C | **BGM** | 単音メロディ (単調なループ) |

- **mixer (R7) は起動時に `$31` で固定**：tone A を切り（ch A はノイズ専用）、tone B/C を有効、noise A を有効、noise B/C を切る。各音の ON/OFF は **音量レジスタ (R8/R9/R10)** で行います（鳴らす時だけ音量 > 0、終わったら 0）。
- 3 ch が別用途なので **BGM 演奏中に発射音・歩行音が重なっても OK**。ノイズ源は 1 つだけですが ch A 専用に回すので奪い合いはありません。

### フレーム駆動（メインループ統合）

```c
sound_init();                 /* 起動時 1 回: PSG 初期化 + BGM 開始 */
for (;;) {
    /* フレームペーシング (deadline。 TIMER.md) */
    while (timer_get() < FRAME_TARGET && g < PACE_SAFETY_CAP) g++;
    timer_start();
    sound_tick();             /* ← BGM 進行 + SE 減衰/リリース。 全 continue の前に */
    ...
    /* BREAK 立ち上がりで発射した瞬間 */   sfx_ball();
    /* 歩行アニメが 1 歩進む瞬間 */         sfx_walk();
}
```

`sound_tick()` はフレームペーシング（`timer_get()` 待ち）の直後（メインループ先頭、どの `continue` よりも前）に置き、**毎フレーム必ず 1 回**呼びます。これで停止中も BGM が進みます。

## 4. C API ([c_sound.h](../src/c_sound.h))

| 関数 | 役割 |
| --- | --- |
| `sound_init()` | PSG 初期化 (mixer 設定 + 全 ch 消音 + ノイズ周期) して BGM を開始 |
| `sound_tick()` | 毎フレーム 1 回。 BGM を 1 ステップ進め、 鳴っている SE を減衰/リリース |
| `sfx_ball()` | ボール発射音 (ch A ノイズ) をトリガ |
| `sfx_walk()` | 歩行音 (ch B 短ブリップ) をトリガ |

## 5. BGM データ形式（ch C 単音）

BGM は「**音程 period と長さ（フレーム）**」の並走 2 配列で、末尾までいくと先頭へループします（`bgm_period[]` / `bgm_frames[]`）。`period = 0` は休符（= ch C 音量 0）。

音程の `period` は **PSGクロック ÷ (16 × 周波数)**。本テンプレは 1 オクターブぶんを定数で持っています（クロック 1.2288MHz 前提、`period ≈ 76800 / 周波数[Hz]`）:

| 音 | 周波数(Hz) | period |
| --- | --- | --- |
| C4 | 261.6 | 294 |
| D4 | 293.7 | 262 |
| E4 | 329.6 | 233 |
| F4 | 349.2 | 220 |
| G4 | 392.0 | 196 |
| A4 | 440.0 | 175 |
| B4 | 493.9 | 156 |
| C5 | 523.3 | 147 |
| (休符) | — | 0 |

`sound_tick()` は現在の音符の残りフレームが尽きたら次の音符の period を R4/R5 に書き、音量を入れます。1 フレーム = 20ms なので、長さ 10 フレーム ≒ 200ms。**曲を差し替えたいときは `bgm_period[]` / `bgm_frames[]` を書き換える**だけです。

## 6. 効果音 (SE)

- **ボール発射音 (ch A)**：`sfx_ball()` で R6（ノイズ周期）をセットし ch A 音量を最大に。`sound_tick()` が**音量減衰テーブル**で数フレームかけて 0 まで落とす（= 「シュッ」というバースト）。発射の瞬間（BREAK 立ち上がりで実際に撃った時）に呼びます。
- **歩行音 (ch B)**：`sfx_walk()` で短い低音の矩形波ブリップ（period 600 ≒ 128Hz）を 2 フレームだけ鳴らしてリリース。歩行アニメの frame が 1 つ進む瞬間に呼ぶので、足の動きと足音が同期します。

いずれも「トリガで状態をセット → `sound_tick()` が残りフレームを数えて自動で消音」する作りで、メインループ側は鳴らしっぱなしを気にしなくて済みます。

## 7. メインループのどこで呼ぶか（まとめ）

| タイミング | 呼ぶもの | 場所 ([c_main.c](../src/c_main.c)) |
| --- | --- | --- |
| 起動時 1 回 | `sound_init()` | `timer_init()` / `timer_start()` の直後 |
| 毎フレーム | `sound_tick()` | フレームペーシング（`timer_get()` 待ち）の直後（全 `continue` の前） |
| ボール発射時 | `sfx_ball()` | `sub_draw_ball(...)` で発射した直後 |
| 1 歩進む時 | `sfx_walk()` | 歩行アニメの frame を進めるブロック内 |

## 8. 拡張アイディア

- **ハードウェアエンベロープ (R11-R13)**：音量をソフトで減衰させる代わりに PSG のエンベロープで自動減衰させると、`sound_tick()` の負荷が減り音の立ち上がり/減衰も滑らかになる。
- **BGM を和音に**：ch A/B も BGM に回して 2〜3 声の和音にできる。ただし SE と ch を奪い合うので、「SE が鳴る瞬間だけ一番低優先の声部を一時的に SE へ貸す（チャンネルスティール）」優先度制御が要る。まず単音で動かしてから拡張するのが安全。
- **テンポ可変**：音長（フレーム数）に係数をかければ全体の速さを変えられる。
- **複数 BGM / ジングル**：`bgm_period[]` を複数用意して切り替える、面クリア時に短いジングルを挿す、など。

## 9. エミュ/実機での検証ポイント（★）

1. **音高**：PSGクロックの実値が前提（1.2288MHz）とずれると音程がずれる。§5 の `period` 表を実測に合わせて調整。
2. **mixer / I/O ポート**：R7 の I/O ポート方向ビット（bit6/7）は入力に倒している。機種で支障が出る場合はここを見直す。
3. **ノイズの音色**：R6 のノイズ周期で発射音の質感が変わる。好みに合わせて調整。
