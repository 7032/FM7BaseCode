# マウスカーソルデモ (FM-7 / FM77AV / AV40系)

マウスカーソルと、画面最上部のタイトル表示、その下の 4 つの画面モード
切替ボタン、現在のマウス方式表示、左右ボタンのインジケータを備えた
デモです。マウスの読み取り方式は**バスマウス**と**インテリジェント
マウス**の両方に対応します。6809 アセンブラ 2 本 (メインCPU側 /
サブCPU側) で構成され、F-BASIC を使わない 2D ディスク直接ブートで
起動します。FM-7 / FM77AV / FM77AV40 / FM77AV40EX の全機種で同じ
ディスクから起動します。

## 起動

同梱の `mousedemo.d77` を Drive 0 にセットして起動してください。
実機・WebM7・その他のエミュレータでそのまま動作します。

## 操作方法

### マウス方式の切り替え (キー `0` / `1` / `2`)

キーボードの `0` / `1` / `2` で読み取り方式を切り替えます
(**キー操作のみで切り替わり、マウス操作では方式は変わりません**)。
現在の方式は画面に `MOUSE: ...` として常時表示されます。

| キー | 方式 | 読み取り先 |
|------|------|-----------|
| `0` | バスマウス | `$FDE8` |
| `1` | インテリジェントマウス ポート1 | OPN ジョイスティック端子 (`$FD15`/`$FD16`) |
| `2` | インテリジェントマウス ポート2 | OPN ジョイスティック端子 (`$FD15`/`$FD16`) |

- キーボードは起動時に `$FD02` へ `#$01` を書いてメインCPUへ解放し、
  メインループで `$FD03` bit0 をポーリングして受けます (割込みベクタは
  使いません)。
- **バス**は移動量を符号反転してラッチし、ボタンは正論理 (押下=1)。
  **インテリジェント**は移動量を符号反転せずにラッチし、ボタンは負論理
  (0=押下)。デモは両者を共通の内部表現 (右・下が正、ボタン bit4=左/
  bit5=右で押下=1) に正規化してからサブへ渡すため、どちらの方式でも
  同じマウス操作で同じ向きにカーソルが動きます。
- 応答しない方式を選んでもカーソルは中央に静止するだけで暴走しません
  (バスは `$FDE8` の bit6、インテリジェントは 4 読みが全 `$FF` かで判定し、
  移動量 0・ボタン解放とみなします)。FM-7 で FM音源カードが無効のときは
  インテリジェントマウスが無応答になります。

### 画面モードの切り替え (マウス左クリック)

タイトルの下に並ぶ 4 つのボタンを左クリックすると画面モードが
切り替わります。

| ボタン | モード | 対応機種 | 主なレジスタ |
|-------|--------|---------|-------------|
| (1) `640x200 8` | 640x200 8色 | 全機種 (起動時の初期モード) | `$FD12` bit6=0 |
| (2) `320x200 4096` | 320x200 4096色 | FM77AV 以降 | `$FD12` bit6=1 |
| (3) `640x400 8` | 640x400 8色 | AV40系のみ | `$FD12` bit6=0 + `$FD04` bit3=0 |
| (4) `320x200 262K` | 320x200 262144色 | AV40系のみ | `$FD12` bit6=1 + `$FD04` bit4=1 |

- 現在のモードのボタンは**赤枠**で強調されます。
- 機種が対応していないモードのボタンは**暗色** (無効表示) になり、
  クリックしても何も起きません。
- タイトル・ボタンラベルともサブCPU空間の CG ROM (`$D800`-、8x8 フォント、
  ASCII配置) で描画します (ボタンは 320 幅モードでは短縮ラベル `640x8` /
  `320x4K` / `640x400` / `262K`)。
- モード切替時は背景・ボタン・カーソルを全再描画します。カーソル座標は
  新旧モードの解像度比で変換されて引き継がれ、画面上の物理位置が維持
  されます (例: 320x200 の (160,100) は 640x200 では (320,100)、640x400
  では (320,200) に相当)。
- **移動量はモードに応じてスケーリングされます**: 640 幅モードは X 2倍、
  400 ラインモードは Y 2倍 (320x200 系は等倍=基準)。これにより全モードで
  画面上の物理的なカーソル速度が一致します。

### 画面表示

| 領域 | ライン | 内容 |
|------|--------|------|
| タイトル帯 | 0-9 | 黒地の帯 (全幅)。行1-8 に白のタイトル文字列 (CG ROM フォント、中央寄せ) |
| ボタン帯 | 12-27 | モード切替ボタン 4 個 (ラベルは行17-24) |
| 方式表示 | 29-38 | 黒帯 + 行30 に現在の方式 `MOUSE: BUS` / `MOUSE: INTELLIGENT P1` / `MOUSE: INTELLIGENT P2` (白文字) |
| ボタン表示 | 41-52 | `LEFT` / `RIGHT` インジケータ。押下中は白 (点灯)、非押下は暗色 (消灯) で左右個別に表示 |
| 生読み値 | 54/63/72 | 診断用。`BUS` / `P1` / `P2` の 3 系統を毎サンプル読み、各行に生バイト列を16進で表示。インテリジェントはストローブ `reg15` 値も `R15:xx` で併記。選択中の行は行頭に `>` を付けて強調 |
| 操作案内 | 84 | `KEY 0:BUS 1:INT-P1 2:INT-P2` (切替キーの凡例) |
| 背景 | 全画面 | モード別の背景 (縦帯 / グラデーション)。上記はその上に重ね描き |

## 機種の実行時判別

| 判別 | 方法 |
|------|------|
| FM-7 / AV系 | ブート直後 (640モード) にメインCPUが `$FD12` を読む。AV系は bit6=0、FM-7 は未実装のオープンバス ($FF、bit6=1) |
| AV40系 | サブCPUが `$D42F` (VRAM バンクセレクト) へ 0 を書いて読み返す。AV40系のみ `$FC|バンク` が読め、他機種は $FF |

判別結果は機種レベル (0=FM-7 / 1=FM77AV / 2=AV40系) として共有RAMで
メイン・サブが共有し、有効モードマスク (ボタンの有効/無効とヒットテスト
の許可) に使います。FM-7 では `$FD12`・`$FD04`・アナログパレット
(`$FD30`-`$FD34`)・`$D42F` に一切書き込みません。

## ビルド / テスト

ディスクイメージ `mousedemo.d77` は同梱済みで、そのまま起動できます。
ソースを改変して作り直す場合は `make` を実行します
([lwtools](http://www.lwtools.ca/) の `lwasm` と Node.js が必要です。
ディスクイメージの組み立て `mkd77.mjs` は Node.js 上で動作します)。

```
make          # ソースから mousedemo.d77 を生成する
make test     # ヘッドレステストを実行する (下記の環境変数が必要)
make clean
```

アセンブラの場所は `make LWASM=/path/to/lwasm` で指定できます。

### ヘッドレステストの実行方法

テストはエミュレータ core (WebM7) 上でデモをブートして検証します。
実行には次の 2 つを環境変数で与えます。**どちらかが見つからない場合、
テストはエラーにならずスキップして正常終了します。**

| 環境変数 | 内容 |
|---------|------|
| `WEBM7_DIR` | [WebM7](https://github.com/7032/WebM7) のディレクトリ (`core/fm7.js` を含む)。未指定時はリポジトリ直下の `vendor/WebM7` ディレクトリがあればそれを使います |
| `FM7_ROM_DIR` | ROM 一式のディレクトリ。`fm7/` (boot_bas / boot_dos / subsys_c) と `fm77av/` (fbasic30 / subsys_a / subsys_b / initiate) のサブディレクトリを持つこと |

```
WEBM7_DIR=/path/to/WebM7 FM7_ROM_DIR=/path/to/roms make test
```

テストのスクリーンショット (PPM) は環境変数 `MOUSEDEMO_OUTDIR`
(既定 `out/`) へ出力されます。

## ディスク構成 (2D: 40シリンダ x 2サイド x 16セクタ x 256バイト)

| トラック0 | 内容 | ロード先 |
|-----------|------|---------|
| サイド0 セクタ1 | IPL | `$0100` (ブートローダが自動ロードし `$0100` へジャンプ) |
| サイド0 セクタ2-15 | メインプログラム | `$0200`-`$0FFF` (IPL がロード) |
| サイド0 セクタ16 + サイド1 セクタ1-16 | サブCPU用プログラム | `$1000`-`$20FF` (IPL がロード後、サブへ転送) |

ブートローダが読むのはトラック0 サイド0 のセクタ1 (256バイト) だけです。
残りのセクタは IPL 自身が FDC (MB8877 相当、I/O `$FD18`-`$FD1F`) を直接
操作して読み込みます (RESTORE 後、READ SECTOR を発行し `$FD1F` の
DRQ/IRQ をポーリング)。ブートROM/ブートRAM 内のルーチンには依存しない
ため、ROM の版差異や機種差 (FM-7 の DOS モードブート / FM77AV系の
ブート) に影響されません。

## 動作の流れ

1. **IPL** — 自前の FDC ルーチンでトラック0 の両面 31 セクタを `$0200`- へ
   ロードし本体へジャンプ
2. **機種判別 (メイン)** — `$FD12` の読み値で FM-7 / AV系 を判別
3. **OPN ポート方向設定 (メイン)** — インテリジェントマウスの読み出しに
   先立ち、OPN のモードレジスタ (reg7) を `$FD15`/`$FD16` 経由で書き、ポートB
   (reg15) を出力・ポートA (reg14) を入力へ設定する (内蔵音源は全チャネル停止)。
   ポート選択線とストローブ線はいずれもポートB のピンにあり、ポートB を出力に
   切り替えていないと reg15 への書込みが物理ピンを駆動できず、ポート選択も
   ストローブによるフェーズ送りも成立しない。全機種で起動時に毎回明示設定する
4. **パレット** — デジタルパレット (`$FD38`-`$FD3F`) を恒等化 (全機種)。
   AV系のみアナログパレット (`$FD30`-`$FD34`) を「添字=色値」の恒等マップで
   4096 エントリ初期化 (添字 i → G=(i>>8)&15, R=(i>>4)&15, B=i&15)
5. **サブCPUプログラム転送・起動** — サブモニタの正規コマンド `$3F`
   (バイトコードインタプリタ) を使う。このコマンド体系は FM-7 と FM77AV系で
   共通:
   - 共有RAM (メイン `$FC80`-`$FCFF` = サブ `$D380`-`$D3FF`) はデュアルポート
     調停のため、メイン側からはサブCPUを HALT (`$FD05` bit7) している間しか
     読み書きできない。全アクセスを HALT/RUN で包む
   - コマンドバイト (サブ `$D382`) に `$3F` を書くと、サブモニタが
     サブ `$D38B` からのバイトコード (`$90`=終了 / `$91` src,dst,len=転送 /
     `$92` addr=分岐 / `$93` addr=呼出) を実行する
   - ペイロード 96 バイトを共有RAM に置き `$91` でサブRAM `$C100`- へ
     コピーさせる、を 24 チャンク (計2304バイト) 繰り返す。完了検知は
     「バイトコード末尾でシーケンス値を ack セルへ転写」+「コマンドバイトが
     0 に戻る」の二重確認
   - 最後に `$93` で `$C100` を呼び出してデモを起動 (制御は戻さない)
   - モニタ内部のスタック位置等、ROM 版に依存する番地は一切仮定しない
6. **機種レベル確定 (サブ)** — サブが `$D42F` の読み返しで AV40系を判別し、
   機種レベルを共有RAMへ報告。メインはそれを有効モードマスクに変換
7. **背景+タイトル+ボタン+ステータス描画 (サブ)** — 現モードのレイアウトで
   背景・タイトル帯とタイトル文字列・ボタン・ラベル・方式表示・左右ボタン
   インジケータ・カーソルを描画
8. **キーボード + マウス (メイン)** — メインループの各周回で、まず
   `$FD03` bit0 をポーリングしてキー入力を確認し、`0`/`1`/`2` なら方式を
   切り替える。続いて選択中の方式でマウスを 1 サンプル読み、移動量
   (右・下が正) とボタン (bit4=左/bit5=右, 押下=1) を得る。
   - **バスマウス** (`$FDE8`): 下位2ビット非ゼロの書込みで移動量をラッチ
     → 4 回読み出しで X下位/X上位/Y下位/Y上位 のニブルを取得。ラッチ値は
     右・下が正 (2の補数)。ボタンは bit5-4 に正論理 (押下=1)。
     **読み値 bit6=1 はマウス I/F 非搭載とみなし、そのポーリングを無視する**
   - **インテリジェントマウス** (OPN ジョイスティック端子): `$FD15`/`$FD16`
     で reg15 (ポートB出力) のストローブ線 (ポート1=bit4/ポート2=bit5) を
     4 回反転させ、各エッジ後に reg14 (ポートA入力) を JOYSTICK コマンド
     (`$09`) で読む。位相1=X上位/2=X下位/3=Y上位/0=Y下位。移動量は符号
     反転なしでラッチされるため右・下を正にするよう符号を揃え、ボタンは
     負論理 (0=押下) を反転して正規化する。**4 読みが全 `$FF` なら無応答
     (FM音源カード無効など) とみなし移動量0・ボタン解放**とする
   - **移動量はモードに応じてスケーリングする**: 640 幅モードは X を 2 倍、
     400 ラインモードは Y を 2 倍にして積算する (320x200 系は等倍=基準)
   - 座標・ボタン・モード・方式のいずれかが変化したらサブCPUを一時停止して
     共有RAM のメールボックス (座標16ビット x2 + ボタン + モード + 方式) を
     書き、更新カウンタを進める
9. **ヒットテスト (メイン)** — 左ボタンの押下エッジで、カーソル座標が
   ボタン帯 (ライン 12-27) のどのボタンに載っているかをレイアウト表で判定。
   有効モードなら `$FD12` (+AV40系は `$FD04`) を書き替え、カーソル座標を
   解像度比で新モードの座標系へ変換して物理位置を維持し、モード番号を
   含めてメールボックス送信 → サブが全再描画
10. **カーソル + ステータス (サブ)** — メールボックスの更新カウンタを監視し、
    変化したら旧位置を XOR で消去 → 新位置へ XOR で描画 (静止中は再描画
    しないためちらつかない)。左ボタン押下中は青チャネルのみ XOR し、カーソル
    色が変わる。方式が変わったら方式表示を、ボタン状態が変わったら左右
    インジケータを、それぞれ変化時のみ再描画する

## モード毎の VRAM レイアウト (サブCPUから見た配置)

| モード | バイト/ライン | プレーン構成 |
|--------|-------------|-------------|
| 640x200 8色 | 80 | B=`$0000` R=`$4000` G=`$8000` (各 16000 バイト使用) |
| 320x200 4096色 | 40 | 12 サブプレーン: ページ0 (`$D430` bit5=0) に B3/B2/R3/R2/G3/G2、ページ1 に B1/B0/R1/R0/G1/G0 (各 `$0000`/`$2000`/`$4000`/`$6000`/`$8000`/`$A000`) |
| 640x400 8色 | 80 | `$D42F` バンク 0/1/2 = B/R/G。各プレーン 32000 バイトがサブ窓 `$0000`-`$7FFF` に見える (`$8000`-`$BFFF` は無効) |
| 320x200 262144色 | 40 | 18 サブプレーン: `$D42F` バンク 0 = ビット5/4、バンク 1 = ビット3/2、バンク 2 = ビット1/0。各バンク内は 4096色と同じ 6 面配置 (B/R/G 各 2 面) |

- 640x400 と 262144色 は AV40系専用。262144色はパレットを経由しない
  直接発色 (B/R/G 各 6 ビット)、640x400 はデジタルパレット 8 色
- 背景: 8色モードは縦帯 8 色、4096色は 赤=X方向・緑=Y方向・青=斜め の
  グラデーション、262144色は 6 ビット精度のグラデーション
  (B=バイト列+(y>>3)、R=バイト列、G=y>>2)

## ヘッドレステストの検証項目

### デモ全体の検証 (`test_mousedemo.mjs`)

機種マトリクス: **FM-7 / FM77AV / FM77AV40 / FM77AV40EX**。期待値モデル
(背景 + タイトル帯/タイトル文字列 + ボタン枠/地/CGフォントラベル +
XOR カーソル) を JS 側で完全再現し、各モードの**全プレーンをバイト単位で
厳密照合**します。

- ブート・機種レベル判別 (0/1/2)・初期 640x200 8色・デジタルパレット恒等
- 初期画面 (背景 + タイトル + ボタン + カーソル) の全プレーン一致、
  静止可視率 100%
- **タイトル描画**: タイトル文字列がタイトル領域の VRAM 非ゼロ +
  CG ROM の期待グリフと完全一致すること (全モードで検証)
- ポインタ移動でカーソルが右・下へ移動し、旧位置が消去されること
- **移動量スケーリング**: 同一デルタ注入で全モードの物理移動量が一致する
  こと (640x200 は X 2倍、640x400 は X/Y とも 2倍、320x200 系は等倍) を
  モード毎の期待座標で厳密判定
- **モード切替時の位置引継ぎ**: 切替直後のカーソルが解像度比で変換した
  座標に表示されること (全切替経路で検証)
- FM-7: 無効ボタン(2)(3)(4) のクリックで何も変わらないこと
- FM77AV: ボタン(2) → 4096色 (アナログパレット恒等含む) → ボタン(1) で復帰。
  無効ボタン(3)(4) のクリックで何も変わらないこと
- AV40系: ボタン(3) → 640x400 (Y>199 へのカーソル移動含む) →
  ボタン(4) → 262144色 → ボタン(1) で復帰。全モードで全プレーン一致
- 左ボタン押下で青チャネルのみ反転 (カーソル色変化)、解放で復元
- **マウス方式のキー切替**: キー `0`/`1`/`2` を注入し、方式表示が全プレーン
  で期待どおり更新されること。**各方式でデルタ注入によりカーソルが実際に
  動くこと**。マウス操作では方式が変わらないこと
- **左右ボタンインジケータ**: 左右ボタンの 4 通りの組合せで
  `LEFT`/`RIGHT` の点灯/消灯が左右個別に反映されること
- **キー入力の頑健性**: ブレークコード有効時に離鍵しても方式が保持される
  こと、同一キーのオートリピート (同一方式の再選択) が無反応であること、
  `0`..`2` 以外の範囲外キーが無視されること
- FM-7 (FM音源カード無効): インテリジェントを選んでもカーソルが中央静止・
  暴走せず、バスへ戻すと再び動くこと
- マウス未接続 / I/F 非搭載相当 (`$FDE8` 読みが常に `$FF`) でも中央静止

### 接続モードとプロトコルの単体検証 (`test_mousemode.mjs`)

OPN レジスタ (`$FD15`/`$FD16` の実 I/O 手順) と `$FDE8` の読み書きで
エミュレータ core を直接駆動して検証します (`make test` に含まれる)。

- 接続モード `setMouseMode()`: `none` / `bus` / `intel1` / `intel2` の排他
  (非選択方式は未接続応答)、切替時の位相・ラッチ・累積リセット、不正値の丸め
- インテリジェントマウス: フェーズ遷移・ラッチとクリア・2の補数の符号・
  int8 クランプ・ボタンとトリガマスク・ポート選択不一致フォールバック・
  タイムアウトリセット・`none` 時の不活性
- 既存 API 互換: `setMouseEnabled(true)` が `bus` 相当、`setMousePort()` の
  モード追従、`reset()` での接続モード保持
- 機種マトリクス: FM77AV / FM77AV40 / FM77AV40EX / FM-7 (FM音源カード
  有効構成)。FM-7 のカード無効構成では OPN ゲートにより不応答となることも
  検証
